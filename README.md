# egg-synthgen

Synthetic EEG dataset generator for training EEG source localization models.

## Overview

This package generates large-scale synthetic EEG datasets with full ground-truth source activity, designed for training deep neural networks, specifically diffusion/flow-matching models. Every sample generated through this package, follows an explicit, fully controlled generative model:

```
Y = R( G · (S_active + α · S_bg) ) + E_sensor + A_artifact
```

where:

- `Y` is the generated scalp EEG (C channels, T time samples)
- `G` is the leadfield matrix (CxN), drawn from a precomputed bank
- `S_active` is the neural source signal (NxT), generated by the source backend
- `S_bg` is background brain activity (NxT), also from the source backend
- `α` is a scale factor achieving the target SNIR
- `R(·)` is the reference operator (average reference, fixed channel, or none)
- `E_sensor` is sensor noise (white Gaussian, 1/f colored, or empirical resting-state)
- `A_artifact` is optional artifact injection (ocular, muscular, line noise, bad channel dropout)

Every parameter that controls this model is recorded as sample metadata in `metadata.jsonl`, enabling full reconstruction of any sample from its `Scenario` record.

## Quick Start

### Step 1: Prepare banks (run once)

Before generating any data, you need to build the **anatomy**, **leadfield**, and **montage banks**. These are precomputed forward models derived from MNE-Python's built-in head models.

```bash
# MNE-based anatomies (fsaverage + mne_sample). Skips nyhead unless --nyhead-mat
# is supplied. Use --anatomy fsaverage ... to restrict to a subset.
python scripts/prepare_banks.py --config config/default.yaml

# Validate what was built (shape / physics / cross-coherence checks)
python scripts/validate_banks.py --banks-dir banks/
```

This creates the following directory structure under `banks/`:

```
banks/
  anatomy/
    fsaverage/
      source_space.npz          # geometry: vertex coords, adjacency, hemisphere
      parcellations/
        desikan_killiany.npz    # integer label per vertex + region_labels
        destrieux.npz           # (optional) finer cortical atlas
        deepsif_994.npz         # (optional) literature-dense atlas
        schaefer_400.npz        # (optional) functional atlas
  leadfield/
    fsaverage/
      standard_1005_32/
        standard.npz        # G matrix (32xN)
      standard_1005_64/
        standard.npz        # G matrix (64xN)
      ...                   # one directory per montage
  montage/
    standard_1005_32.npz    # electrode coords + channel names
    standard_1005_64.npz
    ...
  connectivity/
    desikan_killiany.npz    # built separately, see "Connectivity bank"
  atlases/                  # external atlases fetched by scripts/fetch_atlases.py
    deepsif/
      fs_cortex_20k_region_mapping.mat
      connectivity_998.zip
```

Geometry (vertex coords, adjacency, hemisphere) lives in one file per anatomy. Parcellations live in separate per-scheme files so multiple atlases can coexist on the same geometry. At generation time, only the scheme named by `anatomy_bank.parcellation_scheme` is loaded.

Bank preparation for `fsaverage` with all 7 montages takes approximately 15-30 minutes depending on hardware. `nyhead` is opt-in: download the NY Head MATLAB file (sa_nyhead.mat) and pass `--nyhead-mat /path/to/sa_nyhead.mat` in addition to the default flags.

`validate_banks.py` runs four sections — anatomy geometry, montage coordinates, leadfield column-norm statistics, and cross-coherence of `G`'s dimensions against the anatomy/montage files — and exits with status 1 if any check fails. Pass `--plot` to visually inspect source-space + electrode alignment.

### Step 2: Generate a dataset

```bash
python scripts/generate_dataset.py --config config/default.yaml
```

Output is written to `data/generated/` (configurable via `writer.output_dir`). Progress is printed every 100 samples.

### Step 3: Validate the dataset

```bash
python scripts/validate_dataset.py data/generated
```

Prints group structure, per-group sample counts, EEG shapes, SNIR statistics, and a pass/fail report.

## Bank Preparation

Banks are precomputed once and reused across all dataset generations. Three types of banks are required.

### Anatomy bank

Geometry and parcellations are stored in separate files so multiple parcellations can share one source space.

**Geometry** — `banks/anatomy/{anatomy_id}/source_space.npz`

Fields: `vertex_coords` (Nx3, mm), `adjacency_data` / `adjacency_indices` / `adjacency_indptr` / `adjacency_shape` (sparse CSR cortical neighbourhood graph), `hemisphere` (N, values in {0,1}).

Default spacing is MNE `ico5` ≡ fsaverage5 (10 242 vertices per hemisphere, 20 484 total — matching DeepSIF's reference source space).

**Parcellations** — `banks/anatomy/{anatomy_id}/parcellations/{scheme}.npz`

Fields: `parcellation` (N, int parcel id per vertex, dense 0..R-1), `region_labels` (R, string), `scheme` (string).

Supported schemes and their origin are listed under [Parcellations](#parcellations).

### Leadfield bank

Stores the G matrix (CxN, float32) for each (anatomy, montage, conductivity) combination. The file path encodes all three axes:

```
banks/leadfield/{anatomy_id}/{montage_id}/{conductivity_id}.npz
```

The `leadfield_id` used throughout the codebase is `{anatomy_id}__{montage_id}__{conductivity_id}`.

### Montage bank

Stores 3D electrode coordinates (Cx3, in mm) and channel names for each montage:

```
banks/montage/{montage_id}.npz
```

Fields: `coords`, `ch_names`.

### Parcellations

`prepare_banks.py` takes one or more `--parcellation <scheme>` flags (default: `desikan_killiany`) and writes one file per scheme under `banks/anatomy/{anatomy_id}/parcellations/`.

| Scheme            | # regions | Source (MNE annot / external)                  |
| ----------------- | --------- | ---------------------------------------------- |
| `desikan_killiany`| 68        | `aparc` — shipped with `mne.datasets.fetch_fsaverage` |
| `destrieux`       | 148       | `aparc.a2009s` — shipped with MNE              |
| `hcp_mmp1`        | 360       | `HCPMMP1` (Glasser) — `mne.datasets.fetch_hcp_mmp_parcellation` |
| `deepsif_994`     | 994       | `fs_cortex_20k_region_mapping.mat` from [bfinl/DeepSIF](https://github.com/bfinl/DeepSIF) |
| `schaefer_400`    | 400       | `Schaefer2018_400Parcels_17Networks` — [ThomasYeoLab/CBIG](https://github.com/ThomasYeoLab/CBIG) |
| `schaefer_1000`   | 1000      | same as above, 1000-parcel variant             |

`desikan_killiany`, `destrieux`, `hcp_mmp1` work out of the box once `prepare_banks.py` has run. `deepsif_994`, `schaefer_400`, `schaefer_1000` need one extra fetch step:

```bash
# DeepSIF: 994-region mapping + Hagmann 998 connectome
python scripts/fetch_atlases.py --deepsif

# Schaefer 400 + 1000 into $SUBJECTS_DIR/fsaverage/label/
python scripts/fetch_atlases.py --schaefer --n-parcels 400 1000
```

Then re-run `prepare_banks.py` with the desired schemes, e.g.:

```bash
python scripts/prepare_banks.py --config config/default.yaml \
    --parcellation desikan_killiany deepsif_994 schaefer_400
```

The DeepSIF path expects the source space to be `ico5` (20 484 vertices), matching DeepSIF's `fsaverage5` reference. It is validated at load time and fails loudly otherwise.

### Connectivity bank

Structural connectivity is optional (required only for the TVB backend). One file per parcellation scheme:

```
banks/connectivity/{scheme}.npz
```

Fields: `weights` (RxR, symmetric, zero-diagonal), `tract_lengths` (RxR, mm), `region_centers` (Rx3, mm), `region_labels` (R, string), `scheme` (string).

See [TVB Source Backend](#tvb-source-backend) for how to build it.

| ------------------- | -------- | --------------------- |

## Generating a Dataset

### Basic usage

```bash
python scripts/generate_dataset.py --config config/default.yaml
```

### Custom configuration

Create a YAML override file for your experiment and pass it as `--config`:

```yaml
# config/scenarios/focal_only.yaml
n_samples: 10000
global_seed: 99

scenario_plan:
  prior_family_weights:
    local_contiguous: 1.0
    network_aware:    0.0
    state_dependent:  0.0
    broad_random:     0.0
    tvb_stub:         0.0

writer:
  output_dir: data/focal_only
```

```bash
python scripts/generate_dataset.py --config config/scenarios/focal_only.yaml
```

### Programmatic usage

```python
from synthgen.config import GenerationConfig
from synthgen.pipeline_runner import PipelineRunner

config = GenerationConfig.from_yaml("config/default.yaml")
runner = PipelineRunner(config)
runner.run()
```

### Using individual components

You can use any stage of the pipeline independently:

```python
import numpy as np
from synthgen.config import GenerationConfig
from synthgen.scenario.sampler import ScenarioSampler
from synthgen.banks.anatomy import AnatomyBank
from synthgen.banks.leadfield import LeadfieldBank
from synthgen.banks.montage import MontageBank
from synthgen.sources.priors.local_contiguous import LocalContiguousPrior
from synthgen.sources.sereega_backend import SEREEGABackend
from synthgen.acquisition.pipeline import AcquisitionPipeline

config = GenerationConfig.from_yaml("config/default.yaml")
rng = np.random.default_rng(1234)

# Sample a scenario
sampler = ScenarioSampler(config)
scenario = sampler.sample(rng)

# Load banks
anatomy_bank = AnatomyBank(config.anatomy_bank)
leadfield_bank = LeadfieldBank(config.leadfield_bank)
montage_bank = MontageBank(config.montage_bank)

source_space = anatomy_bank.load(scenario.anatomy_id)
leadfield = leadfield_bank.load(scenario.leadfield_id)
montage = montage_bank.load(scenario.montage_id)

# Fill spatial/temporal parameters
prior = LocalContiguousPrior()
scenario_rng = np.random.default_rng(scenario.seed)
scenario = prior.sample(scenario, source_space, scenario_rng)

# Generate source signals
backend = SEREEGABackend(config)
source_activity, background_activity = backend.generate(scenario, source_space, scenario_rng)

# Run acquisition pipeline
pipeline = AcquisitionPipeline(config)
sample, qc = pipeline.run(
    scenario, source_space,
    source_activity, background_activity,
    leadfield, montage.coords, montage.ch_names,
    scenario_rng,
)

print(f"EEG shape: {sample.eeg.shape}")       # (C, T)
print(f"QC passed: {qc.passed}")
print(f"SNIR: {sample.snir_measured_db:.1f} dB")
```

## Dataset Format

The output directory has this structure:

```
data/generated/
  data.zarr/
    fsaverage__standard_1005_64/    ← one group per (anatomy, montage) combination
      eeg               (N, C, T)   float32   - final EEG after full pipeline
      source_activity   (N, V, T)   float32   - original ground thruth source activation
      source_support    (N, V)      bool      - active source vertex mask
      snir_db           (N,)        float32   - signal-to-noise+interference ratio (dB)
      snr_sensor_db     (N,)        float32   - sensor-level SNR (dB)
      active_area_cm2   (N,)        float32   - total activated cortical area
    fsaverage__standard_1005_32/
      eeg               (N, 32, T)  float32
      ...
    .zattrs                         - root attributes: n_samples, schema_version
  metadata.jsonl                    - one JSON line per sample: full Scenario as dict
```

where:

- N -> Number of samples per group (anatomy, montage)
- C -> Channel count
- V -> Total vertex (actve and non active)
- T -> Time samples (dependent on the sfreq and window size)

**Why grouped by anatomy+montage?** Different montages have different channel counts `C` (21-256) and different anatomies have different vertex counts `V`. Grouping ensures arrays within each group have fixed dimensions, enabling efficient chunked I/O.

**Why JSONL for metadata?** The full `Scenario` dataclass (including all spatial, temporal, noise, and artifact parameters) is serialized as JSON. This makes per-sample provenance fully queryable without loading zarr arrays.

**Note on source_activity:** Raw source-space activity (NxT per sample) is intentionally not stored - at ~4 MB per sample, storing 100K samples would require ~400 GB. It can be regenerated from the `seed` and `config_hash` fields in `metadata.jsonl`.

### Accessing data

```python
import zarr
import json
import numpy as np

# Open the zarr store
store = zarr.open("data/generated/data.zarr", mode="r")

# List all anatomy+montage groups
for key in store.group_keys():
    grp = store[key]
    print(f"{key}: {grp['eeg'].shape}")  # e.g. fsaverage__standard_1005_64: (50000, 64, 500)

# Load EEG for a specific group
eeg = store["fsaverage__standard_1005_64"]["eeg"][:]          # (N, 64, 500)
support = store["fsaverage__standard_1005_64"]["source_support"][:]  # (N, V)
snir = store["fsaverage__standard_1005_64"]["snir_db"][:]     # (N,)

# Load metadata for all samples
with open("data/generated/metadata.jsonl") as f:
    scenarios = [json.loads(line) for line in f]

# Filter by prior family
focal_indices = [i for i, s in enumerate(scenarios) if s["prior_family"] == "local_contiguous"]
```

### Schema version

The zarr root `.zattrs` contains:

```json
{
  "n_samples": 100000,
  "schema_version": "1"
}
```

## Configuration Reference

All parameters are set in a YAML file (default: `config/default.yaml`). Every field has a validated Pydantic schema.

### `anatomy_bank`

```yaml
anatomy_bank:
  bank_dir: banks/anatomy         # directory containing anatomy subdirectories
  anatomy_ids:                    # anatomies available for scenario sampling
    - fsaverage
    - mne_sample
    - nyhead
  parcellation_scheme: desikan_killiany  # which parcellation to load per anatomy
                                          # (see "Parcellations" for supported values)
```

### `leadfield_bank`

```yaml
leadfield_bank:
  bank_dir: banks/leadfield       # directory containing leadfield files
  conductivity_ids:               # conductivity model identifiers
    - standard
```

### `montage_bank`

```yaml
montage_bank:
  bank_dir: banks/montage         # directory containing montage files
  montages:
    - {name: standard_1005_64, n_channels: 64, split_role: core}
    - {name: standard_1005_21, n_channels: 21, split_role: ood}
    # split_role: "core" → train/val/test splits
    # split_role: "ood"  → OOD evaluation only
```

### `connectivity_bank`

```yaml
connectivity_bank:
  bank_dir: banks/connectivity    # directory containing {scheme}.npz files
  schemes:                        # schemes available to the TVB backend
    - desikan_killiany
```

Only used when `backend: tvb`. The entry for `tvb.connectivity_scheme` must appear in this list.

### `scenario_plan`

Controls the marginal distribution of generated samples.

```yaml
scenario_plan:
  prior_family_weights:           # must sum to 1.0
    local_contiguous: 0.30        # spatially clustered sources
    network_aware:    0.25        # sources within same cortical parcel
    state_dependent:  0.20        # brain state drives signal type + frequency
    broad_random:     0.15        # random placement anywhere in brain
    tvb_stub:         0.10        # reserved for TVB backend (uses broad_random fallback)

  n_sources_weights:              # number of simultaneous active sources
    weights:
      1: 0.20                     # must sum to 1.0
      2: 0.25
      3: 0.30
      4: 0.25

  difficulty_weights:             # controls SNIR range and patch extent
    easy:   0.40                  # must sum to 1.0
    medium: 0.35
    hard:   0.25

  ood_fraction: 0.10              # fraction of samples assigned to OOD montages
```

### `temporal`

```yaml
temporal:
  sfreq: 500.0                    # sampling frequency in Hz
  window_s: 1.0                   # epoch duration in seconds
  signal_families:                # active signal types (uniform random selection)
    - erp
    - oscillatory_burst
    - ar_correlated
    - spike_interictal
```

Derived: `n_samples_per_window = int(sfreq * window_s)` = 500 time points per epoch.

### `background`

```yaml
background:
  families:                       # source-space background activity families
    - nmm_background              # TVB neural-mass-model output (used by tvb backend)
    - colored_1f                  # 1/f pink noise (sereega default)
    - ar_background               # AR(1) per-vertex correlated noise
  weights: [0.4, 0.4, 0.2]        # must sum to 1.0, same length as families
```

Background is generated in source space at the same (N, T) shape as `source_activity` and scaled after forward projection to hit the target SNIR.

### `noise`

```yaml
noise:
  families:
    - white_gaussian
    - colored_1f
    - empirical_resting
  weights: [0.5, 0.3, 0.2]       # must sum to 1.0
  snr_core_range_db: [0.0, 20.0] # sensor SNR range for core (and OOD) samples
  snr_hard_range_db: [-5.0, 30.0] # SNIR range for hard difficulty (wider, includes negative)
```

Note: Sensor SNR range (`snr_core_range_db`) is always used for the noise step, regardless of difficulty. Difficulty is captured via SNIR (signal-to-noise+interference ratio) which controls background activity scaling.

### `artifacts`

```yaml
artifacts:
  families:
    - ocular
    - muscular
    - line_noise
    - bad_channel_dropout
  artifact_prob: 0.30             # probability of injecting an artifact per sample
```

When an artifact is injected, a family is chosen uniformly from the list.

### `reference`

```yaml
reference:
  scheme: average                 # "average", "fixed", or "none"
  fixed_channel: null             # channel name for "fixed" scheme (e.g. "Cz")
```

### `qc`

```yaml
qc:
  min_valid_channels: 10          # minimum channels with std > 1e-6
  max_snir_deviation_db: 5.0      # (reserved, not currently enforced)
  min_inter_source_distance_mm: 10.0  # minimum pairwise distance between sources
```

Samples failing QC are silently skipped; the runner continues until `n_samples` passing samples are collected.

### `writer`

```yaml
writer:
  output_dir: data/generated      # output directory for zarr store + metadata.jsonl
  chunk_size: 256                 # number of samples buffered before flushing to zarr
```

### `tvb`

Only consulted when `backend: tvb`. See [TVB Source Backend](#tvb-source-backend) for end-to-end usage.

```yaml
tvb:
  model: jansen_rit                     # "jansen_rit" | "generic_2d_oscillator" | "wilson_cowan"
  connectivity_scheme: desikan_killiany # must exist in banks/connectivity/
  global_coupling: 0.1                  # TVB "a" / long-range coupling strength, >= 0
  noise_sigma: 1.0e-5                   # stochastic integrator noise std, >= 0
  integrator_dt_ms: 1.0                 # TVB integration step, > 0 (ms)
  warmup_s: 5.0                         # transient discarded before each sample window, >= 0
  reservoir_duration_s: 60.0            # reserved for future reservoir-pooling strategy, > 0
  reservoir_size: 100                   # reserved, > 0
  stimulus_amplitude: 1.0               # region-level stimulus scaling, >= 0
  conduction_speed_mm_per_ms: 3.0       # axonal conduction speed, > 0
```

### Top-level fields

```yaml
backend: sereega                  # "sereega" (fast, synthetic) | "tvb" (NMM simulation)
n_samples: 100000                 # total QC-passing samples to generate, > 0
n_workers: 4                      # (reserved, runner is currently sequential), > 0
global_seed: 42                   # master RNG seed for full reproducibility
```

## Architecture

### Pipeline stages

The generation pipeline is a sequential chain of pluggable stages, each defined as an abstract base class:

```
ScenarioSampler
    ↓  Scenario (discrete params: anatomy, montage, prior, n_sources, difficulty, ...)
ActivationPrior.sample()
    ↓  Scenario (+ spatial params: seed vertices, patch extents, frequencies, onsets, ...)
SourceGeneratorBackend.generate()
    ↓  source_activity (NxT), background_activity (NxT)
ForwardProjector.project()          ← shared G matrix from LeadfieldBank
    ↓  signal_eeg (CxT), bg_eeg (CxT)
[background scaling to target SNIR]
    ↓  clean_eeg (CxT)
SensorNoiseEngine.apply()
    ↓  noisy_eeg (CxT)
ArtifactEngine.apply()              ← optional, with probability artifact_prob
    ↓  noisy_eeg (CxT)
ReferenceOperator.apply()
    ↓  final_eeg (CxT)
check_sample()
    ↓  QCResult
DatasetWriter.write()               ← only for QC-passing samples
```

### Key design decisions

**Shared leadfield bank.** The leadfield matrix `G` is always drawn from the precomputed bank. No source backend computes its own forward model. This ensures that differences between samples generated by different priors or backends are attributable only to source dynamics - never to differences in the head model. It also enables forward-consistency losses in domain adaptation.

**Scenario-first sampling.** Discrete parameters (anatomy, montage, prior family, difficulty, n_sources, signal family) are sampled first by `ScenarioSampler`. Continuous parameters (vertex positions, patch sizes, temporal onsets, frequencies) are sampled by the prior, conditioned on the discrete choices. This prevents the common error of fully independent parameter sampling that produces degenerate or physically implausible combinations.

**Per-scenario seeding.** Each scenario has its own `seed` field, drawn at sampling time. All downstream stages (prior, backend, acquisition pipeline) receive a `numpy.random.Generator` seeded from `scenario.seed`. The main generation RNG only advances for scenario-level draws. This means any sample can be reproduced exactly from its `seed` and `config_hash`, regardless of the global `n_samples` or run order.

**Grouped zarr storage.** Arrays are grouped by `anatomy_id__montage_id` because different montages have different channel counts (21-256) and different anatomies have different vertex counts. Grouping ensures fixed-dimension arrays within each group, which is required by zarr's chunked storage format.

**TVB integration point.** `TVBSourceGenerator` in `synthgen/sources/tvb_backend.py` is a defined stub. When implemented, it runs NMM simulations at the region level and maps output to the shared vertex source space via the parcellation - it never touches the leadfield. See [Extending the Pipeline](#extending-the-pipeline).

### `synthgen.sample`

Core data structures.

**`SourceSpace`** (dataclass)

Stores the cortical source space loaded from the anatomy bank.

| Field                | Type                  | Description                              |
| -------------------- | --------------------- | ---------------------------------------- |
| `vertex_coords`      | `ndarray (N, 3)`      | Source vertex positions in mm            |
| `adjacency`          | `scipy.sparse (N, N)` | Cortical neighborhood graph              |
| `parcellation`       | `ndarray (N,)`        | Integer parcel label per vertex (0..R-1) |
| `hemisphere`         | `ndarray (N,)`        | 0=left hemisphere, 1=right               |
| `parcellation_scheme`| `str`                 | Scheme id, e.g. `"desikan_killiany"`     |

- `load(scheme: str) -> Connectivity` — `.weights (R, R)`, `.tract_lengths (R, R)`, `.region_centers (R, 3)`, `.region_labels list[str]`, `.scheme str`. Cached per scheme.

**`ScenarioSampler(config: GenerationConfig)`**

- `sample(rng: np.random.Generator) -> Scenario`

Draws from config-weighted distributions. OOD montages always produce `split="ood"`; core montages produce `split="train"`. Raises `ValueError` if no core montages are configured.

### `synthgen.sources`

**`SourceGeneratorBackend`** (ABC)

- `generate(scenario, source_space, rng) -> tuple[ndarray (N, T), ndarray (N, T)]`

Returns `(source_activity, background_activity)`, both float32.

**`SEREEGABackend(config: GenerationConfig)`**

Pure Python implementation. For each active source vertex, generates a signal waveform according to `scenario.signal_family`. Background is 1/f noise across all N vertices, normalized to ~0.1x signal RMS.

**`TVBSourceGenerator(config: GenerationConfig, connectivity: Connectivity)`**

Neural-mass backend built on The Virtual Brain. Runs paired stimulus/baseline simulations and subtracts them on the seed parcels so only stimulus-driven activity remains; the rest of the cortex carries the shared background state. Requires a connectivity bank matching `config.anatomy_bank.parcellation_scheme` and is selected via `backend: tvb` in the config. See [TVB Source Backend](#tvb-source-backend) for full details.

### `synthgen.sources.priors`

**`ActivationPrior`** (ABC)

- `sample(scenario, source_space, rng) -> Scenario`

Mutates `scenario` in-place to fill all spatial and temporal fields, then returns it.

See [Activation Priors](#activation-priors) for a full description of each prior.

### `synthgen.forward.projector`

**`LinearProjector`**

- `project(source_activity: (N, T), leadfield: (C, N)) -> ndarray (C, T)`

Computes `leadfield @ source_activity`. This is the only forward computation in the entire pipeline; all other stages operate in sensor or source space independently.

### `synthgen.acquisition`

**`AcquisitionPipeline(config: GenerationConfig)`**

- `run(scenario, source_space, source_activity, background_activity, leadfield, electrode_coords, ch_names, rng) -> tuple[EEGSample, QCResult]`

Orchestrates the full acquisition chain. Mutates `scenario` in-place (sets `snir_db`, `snr_sensor_db`, `artifact_flags`). Pass a distinct `Scenario` instance per call when running in parallel.

**`ReferenceOperator`** implementations: `AverageReference`, `FixedReference(channel_index)`, `NoReference`

**`SensorNoiseEngine`** implementations: `WhiteGaussianNoise`, `Colored1fNoise(config)`, `EmpiricalRestingNoise`

**`ArtifactEngine`** implementations: `OcularArtifact(config)`, `MuscularArtifact(config)`, `LineNoiseArtifact(config)`, `BadChannelDropout`

**`check_sample(sample: EEGSample, config: QCConfig) -> QCResult`**

See [Quality Control](#quality-control).

### `synthgen.writer`

**`DatasetWriter`** (ABC)

- `write(sample: EEGSample) -> None`
- `finalize() -> None`

**`ZarrWriter(config: WriterConfig)`**

Implements `DatasetWriter`. Buffers samples in memory, flushing to zarr when the buffer reaches `chunk_size`. Supports context manager protocol (`with ZarrWriter(config) as w: ...`).

### `synthgen.pipeline_runner`

**`PipelineRunner(config: GenerationConfig)`**

- `run() -> None`

Runs the full sequential generation loop. At startup it dispatches the source backend based on `config.backend` (`"sereega"` for [`SEREEGABackend`](#synthgensources), `"tvb"` for [`TVBSourceGenerator`](#synthgensources)); when `backend == "tvb"` it also loads the connectivity bank matching `config.anatomy_bank.parcellation_scheme`.

Each scenario is attempted up to 100 times; counters are reported at the end:

- `n_ok` — samples that passed QC and were written.
- `n_skipped_qc` — scenarios that failed QC every attempt and were abandoned.
- `n_skipped_bank` — scenarios that could not even be attempted because a required anatomy/leadfield/montage file was missing (`FileNotFoundError` from any `Bank.load` call). This lets you keep partial banks without crashing the run.

Raises `RuntimeError` if more than `n_samples × 100` total attempts are made without collecting enough QC-passing samples (prevents infinite loops from misconfigured bank paths or QC thresholds).

## Activation Priors

Priors fill the spatial and temporal fields of a `Scenario`. Each prior selects source vertex positions, patch extents, inter-source distances, temporal onsets, and dominant frequencies.

> **TODO (spatial priors).** The current implementation controls patch spatial extent through the number of included vertices. This is density-dependent: the *physical* extent of a patch of K vertices depends on the spacing of the source space (e.g. `ico5` vs. `ico4`), which makes `patch_extents_cm2` values not directly comparable across anatomies or spacings. Planned fix: switch to mm-based geodesic radii on the cortical surface (propagate along the sparse `adjacency` graph weighted by edge length), so that a "5 cm² patch" corresponds to the same physical region regardless of mesh resolution. Tracked for a follow-up iteration; does not affect API compatibility.

### `BroadRandomPrior`

Sources placed uniformly at random across the entire brain. No spatial constraints.

| Parameter          | Range                   |
| ------------------ | ----------------------- |
| Patch extent       | 0.5-10.0 cm² per source |
| Source correlation | 0.0-1.0                 |
| Temporal onset     | 0-0.5 s                 |
| Frequency          | Signal-family-dependent |

Use case: maximum diversity; prevents the model from assuming sources are always focal or network-constrained.

### `LocalContiguousPrior`

Sources placed in a spatially compact cluster. Patch extent is scaled by difficulty.

| Difficulty | Patch extent |
| ---------- | ------------ |
| Easy       | 2-8 cm²      |
| Medium     | 1-4 cm²      |
| Hard       | 0.5-2 cm²    |

| Parameter          | Range                        |
| ------------------ | ---------------------------- |
| Source correlation | 0.0-0.3 (tighter than broad) |
| Temporal onset     | 0-0.4 s                      |

Use case: focal epileptic discharges, small motor activations.

### `NetworkAwarePrior`

Sources constrained to the same cortical parcel (as defined by the anatomy's parcellation). Enforces anatomical network plausibility.

| Parameter          | Range                        |
| ------------------ | ---------------------------- |
| Patch extent       | 0.5-3.0 cm²                  |
| Source correlation | 0.3-0.8 (high, same network) |
| Temporal onset     | 0-0.3 s                      |

Use case: large-scale network activations, default-mode network, sensorimotor rhythms.

### `StateDependentPrior`

Selects a brain state and overrides `scenario.signal_family` and `dominant_frequencies_hz` to match the state's characteristic oscillation.

| Brain state    | Signal family       | Frequency range |
| -------------- | ------------------- | --------------- |
| `alpha`        | `oscillatory_burst` | 8-13 Hz         |
| `mu_beta`      | `oscillatory_burst` | 13-30 Hz        |
| `dmn`          | `ar_correlated`     | 1-4 Hz          |
| `epileptiform` | `spike_interictal`  | 3-5 Hz          |

| Parameter          | Range       |
| ------------------ | ----------- |
| Patch extent       | 1.0-5.0 cm² |
| Source correlation | 0.2-0.7     |
| Temporal onset     | 0-0.3 s     |

Use case: realistic brain-state-specific activations; ensures temporal dynamics match the spatial network.

### `tvb_stub` (BroadRandomPrior fallback)

The `tvb_stub` entry in `scenario_plan.prior_family_weights` is still mapped to `BroadRandomPrior` (see `_PRIOR_REGISTRY` in `synthgen/pipeline_runner.py`). It is *independent of* `backend: tvb`:

- **Priors** decide *which vertices* get seeded and *what the scenario metadata looks like*. `tvb_stub` produces the same broad-random seed selection as `broad_random`.
- **Backends** decide *how the time series on those seeds is generated*. `backend: tvb` runs a Jansen–Rit network on the connectivity bank regardless of which prior picked the seeds.

You can combine them freely: any prior family can drive either the SEREEGA or TVB backend. The `tvb_stub` name is kept so future NMM-informed seed-selection rules can slot in without a config migration.

## Signal Families

Signal waveforms are generated by `SEREEGABackend` for each active source vertex. Each family is parameterized by frequency and onset from the prior.

### `erp`

Event-related potential: a Gaussian-windowed pulse.

$$
\text{amplitude}(t) = A \cdot \exp{\left(-\frac{(t - \text{peak}_t)^2}{2 \cdot\text{width}^2}\right)}
$$

- Peak time: uniform in `[onset + 0.05, onset + 0.2]` s
- Width: uniform in `[0.02, 0.08]` s
- Amplitude: uniform in `[0.5, 2.0]`

### `oscillatory_burst`

Sinusoidal burst with Gaussian envelope at the dominant frequency.

$$
\text{signal}(t) = A \cdot \exp \left( -\frac{(t - center)^2}{2 \cdot \text{width}^2}\right) \cdot \sin(2\pi \cdot \text{freq} \cdot t)
$$

- Burst center: uniform in `[onset + 0.1, onset + 0.3]` s
- Burst width: uniform in `[0.1, 0.2]` s
- Amplitude: uniform in `[0.5, 2.0]`

### `ar_correlated`

AR(1) filtered white noise - a simple model for broadband correlated background-like activity.

```
x(t) = ar_coef · x(t-1) + ε(t),   ε ~ N(0, 1)
```

- AR coefficient: uniform in `[0.7, 0.97]`

### `spike_interictal`

Template-based interictal epileptiform discharge: a negative spike followed by a positive slow wave.

```
signal(t) = -A · exp(-(t - spike_t)² / (2 · spike_w²))      # negative spike
           + 0.5A · exp(-(t - wave_t)² / (2 · wave_w²))      # positive slow wave
```

## TVB Source Backend

The `tvb` backend replaces the SEREEGA-style point-source generators with a [The Virtual Brain](https://www.thevirtualbrain.org/) whole-brain simulation. Both `source_activity` (evoked, masked to the seed parcels) and `background_activity` (spontaneous, whole-brain) are derived from paired per-sample TVB simulations with and without stimulus.

### Installation

```bash
pip install -e ".[tvb]"
```

### Prepare the connectivity bank

The anatomy bank must already contain the target parcellation (see [Parcellations](#parcellations)). Then:

```bash
# DK-68 surrogate from TVB-192 base
python scripts/prepare_connectivity.py --config config/default.yaml \
    --anatomy fsaverage --scheme desikan_killiany --tvb-base 192

# DeepSIF-994 using native 998-region Hagmann-style connectome
python scripts/prepare_connectivity.py --config config/default.yaml \
    --anatomy fsaverage --scheme deepsif_994 \
    --deepsif-dir banks/atlases/deepsif
```

Two code paths:

- **Surrogate** (default): loads one of TVB's bundled connectomes (`--tvb-base` ∈ {66, 68, 76, 96, 192}; default 192) and projects it onto the parcel centroids of the target scheme by nearest-centroid mapping. Symmetrizes and zeroes the diagonal.
- **Native DeepSIF**: triggered automatically when `--scheme deepsif_994` is used and `connectivity_998.zip` exists in `--deepsif-dir`. Projects the 998-region Hagmann-style connectome onto the 994 DeepSIF parcels.

Both paths write the same file format (see [Connectivity bank](#connectivity-bank)). Replace the generated file with a dMRI-derived connectome to use real subject-specific structural connectivity.

### Enable in config

```yaml
backend: tvb
tvb:
  model: jansen_rit          # or generic_2d_oscillator, wilson_cowan
  connectivity_scheme: desikan_killiany
  global_coupling: 0.1
  noise_sigma: 1.0e-5
  integrator_dt_ms: 1.0
  warmup_s: 5.0
  stimulus_amplitude: 1.0
  conduction_speed_mm_per_ms: 3.0
```

### How the decomposition works

For each sample the backend runs two TVB simulations sharing the same noise seed and the same `warmup_s + window_s` duration:

1. **Baseline**: no stimulus. Its final-window output, broadcast from parcel to vertex via `parcellation`, becomes `background_activity`.
2. **Stimulus**: a temporal pattern (derived from `scenario.signal_family`) is injected into the parcels that contain `scenario.seed_vertex_indices`.

The difference `stim − baseline`, restricted to the vertices of those seed parcels, becomes `source_activity`. Every other vertex in `source_activity` is exactly zero.

### Performance

Each sample requires two independent TVB runs of `warmup_s + window_s`, so generation is materially slower than with the SEREEGA backend (hours-to-days for 100k samples on a single CPU). Parallelize via `n_workers`. `window_s` and `warmup_s` dominate the wall clock.

## Noise Models

Sensor noise is applied after the forward projection and background mixing step. The target sensor SNR is drawn uniformly from `snr_core_range_db` for all samples.

### `WhiteGaussianNoise`

Independent Gaussian noise on all channels:

```
noise_std = signal_rms / (10^(snr_db / 20))
noise(t) ~ N(0, noise_std²)
```

### `Colored1fNoise`

Spectrally shaped noise with 1/f power spectrum (pink noise). Generated in the frequency domain:

```
X(f) = W(f) / max(f, 1.0)
```

where `W(f)` is white noise in frequency domain. Scaled to achieve target RMS matching the SNR target.

### `EmpiricalRestingNoise`

Per-channel AR(2) noise approximating resting-state EEG:

```
x(t) = ar1 · x(t-1) + ar2 · x(t-2) + ε(t)
ar1 ~ U(1.0, 1.4),   ar2 ~ U(-0.5, -0.3)
```

This produces a low-frequency-dominant spectrum similar to measured resting EEG.

## Artifact Models

Artifacts are injected with probability `artifact_prob` (default 0.30). When injected, the family is chosen uniformly.

### `OcularArtifact`

Simulates an eye blink: a Gaussian-envelope pulse with maximum amplitude over frontal channels.

- Duration: `blink_width` ∈ [0.05, 0.15] s
- Amplitude: 5-15x signal RMS
- Per-channel weights drawn from U(0, 1), giving realistic spatial variation

### `MuscularArtifact`

High-frequency burst (30-100 Hz) on a random subset of channels.

- Affects 1 to `min(C, max(2, C // 4))` channels
- Burst amplitude: 1-3x signal RMS
- Burst duration: 0.1-0.3 s, Gaussian-windowed

### `LineNoiseArtifact`

Powerline interference at 50 Hz or 60 Hz (chosen uniformly).

- Uniform across all channels
- Amplitude: 0.5-2x signal RMS

### `BadChannelDropout`

Sets 1 to `max(2, C // 8)` randomly selected channels to zero. Simulates electrode disconnection or saturation. Dropped channel names are recorded in `scenario.artifact_flags`.

## Quality Control

Every sample is checked by `check_sample(sample, qc_config)` before being written. Samples that fail QC are counted and skipped; the runner continues until the target count is reached.

**Check 1 - NaN/Inf.** Any non-finite value in the EEG array causes immediate rejection.

**Check 2 - Valid channels.** Channels with standard deviation $\leq$ 1e-6 are considered flat/invalid (e.g. zeroed by `BadChannelDropout`). If the number of valid channels is below `min_valid_channels`, the sample is rejected.

**Check 3 - Inter-source distance.** For multi-source samples, the minimum pairwise Euclidean distance between seed vertices must be $\geq$ `min_inter_source_distance_mm`. Prevents degenerate configurations where two sources are at the same location.

`QCResult` carries:

- `passed: bool`
- `reasons: list[str]` - human-readable failure reasons (empty if passed)

## OOD Splits

The `split` field in `Scenario` is assigned at sampling time, not post-hoc.

| `split` value | Condition                                                           |
| ------------- | ------------------------------------------------------------------- |
| `"train"`     | Core montage, selected ~90% of the time                             |
| `"ood"`       | OOD montage, selected with probability `ood_fraction` (default 10%) |

The 21-channel `standard_1005_21` montage is designated OOD (`split_role: ood`) and will never appear in `split="train"` samples. All core montages (32-256 ch) always appear as `split="train"`.

This design enables evaluation of:

- **Montage generalization:** model trained on 32-256 ch, evaluated on 21 ch
- **Channel-count robustness:** electrode-agnostic architectures

To add additional OOD axes (e.g. anatomy, prior family), change `split_role` in the config or add additional sampling logic to `ScenarioSampler`.

### Adding a new activation prior

1. Create `synthgen/sources/priors/my_prior.py`
2. Implement `ActivationPrior`:

```python
from synthgen.sources.priors.base import ActivationPrior
from synthgen.sample import Scenario, SourceSpace
import numpy as np

class MyPrior(ActivationPrior):
    def sample(
        self,
        scenario: Scenario,
        source_space: SourceSpace,
        rng: np.random.Generator,
    ) -> Scenario:
        N = len(source_space.vertex_coords)

        # Select source vertices
        indices = rng.choice(N, size=scenario.n_sources, replace=False).tolist()
        scenario.seed_vertex_indices = [int(i) for i in indices]
        scenario.patch_extents_cm2 = [float(rng.uniform(1.0, 5.0)) for _ in indices]

        # Compute inter-source distances
        from synthgen.sources.priors._helpers import _pairwise_distances_mm
        scenario.inter_source_distances_mm = _pairwise_distances_mm(
            source_space.vertex_coords, indices
        )

        scenario.source_correlation = float(rng.uniform(0.0, 0.5))
        scenario.temporal_onsets_s = [float(rng.uniform(0.0, 0.4)) for _ in indices]

        from synthgen.sources.priors._helpers import _sample_frequencies
        scenario.dominant_frequencies_hz = _sample_frequencies(
            scenario.signal_family, scenario.n_sources, rng
        )
        return scenario
```

1. Add it to `_PRIOR_REGISTRY` in `synthgen/pipeline_runner.py` and to `prior_family_weights` in your YAML config.

## Reproducibility

Every sample stores enough information to reproduce it exactly:

- **`seed`**: per-scenario RNG seed (int64 from 0 to 2^63-1)
- **`config_hash`**: 8-character SHA256 of the full `GenerationConfig`
- **`scenario_id`**: `s{seed:020d}` - unique, deterministic identifier

The complete `Scenario` object is written to `metadata.jsonl` as a JSON line, capturing every parameter that influences sample generation.

To reproduce a specific sample from its metadata:

```python
import json
import numpy as np
from synthgen.config import GenerationConfig
from synthgen.sample import Scenario
from synthgen.banks.anatomy import AnatomyBank
from synthgen.banks.leadfield import LeadfieldBank
from synthgen.banks.montage import MontageBank
from synthgen.sources.priors.broad_random import BroadRandomPrior  # or whichever prior
from synthgen.sources.sereega_backend import SEREEGABackend
from synthgen.acquisition.pipeline import AcquisitionPipeline

# Load scenario from metadata
with open("data/generated/metadata.jsonl") as f:
    records = [json.loads(line) for line in f]

# Find a specific sample
record = next(r for r in records if r["scenario_id"] == "s00000000042")
scenario = Scenario(**record)
scenario_rng = np.random.default_rng(scenario.seed)

# Reload config
config = GenerationConfig.from_yaml("config/default.yaml")

# Load banks
source_space = AnatomyBank(config.anatomy_bank).load(scenario.anatomy_id)
leadfield = LeadfieldBank(config.leadfield_bank).load(scenario.leadfield_id)
montage = MontageBank(config.montage_bank).load(scenario.montage_id)

# Regenerate source signals
backend = SEREEGABackend(config)
source_activity, background_activity = backend.generate(scenario, source_space, scenario_rng)

# Regenerate EEG
pipeline = AcquisitionPipeline(config)
sample, qc = pipeline.run(
    scenario, source_space,
    source_activity, background_activity,
    leadfield, montage.coords, montage.ch_names,
    np.random.default_rng(scenario.seed),   # reset rng for acquisition step
)
```
